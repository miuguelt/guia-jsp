JDBC Puro

  • ✓ Control total del SQL
  • ✓ Sin dependencias extra
  • ✓ Optimizable al máximo
  • ✗ Mucho código boilerplate
  • ✗ Lento de desarrollar
  • ✗ Errores SQL en runtime

JPA/Hibernate

  • ✓ 70% menos código
  • ✓ SQL generado automático
  • ✓ Errores en compilación
  • ✗ Dependencia Hibernate
  • ✗ Curva de aprendizaje
  • ✗ N+1 queries problem

Analogía Flask: JDBC ≈ db.engine.execute("INSERT...") | JPA ≈ db.session.add(objeto) (SQLAlchemy ORM)
