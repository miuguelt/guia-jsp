El Patrón Modelo-Vista-Controlador

MVC es un patrón de arquitectura de software que separa la aplicación en tres componentes interconectados:

Modelo

Representa los datos y la lógica de negocio. Contiene las entidades (Records) y las clases DAO para acceso a datos.

Producto.java, ProductoDAO.java

Vista

La interfaz de usuario. Muestra los datos al usuario y captura su entrada. Implementada con JSP + JSTL.

lista.jsp, formulario.jsp

Controlador

Recibe las peticiones del usuario, coordina el modelo y selecciona la vista apropiada. Implementado con Servlets.

ProductoServlet.java

Flujo de una Petición MVC

1

Usuario hace clic en "Crear Producto"

2

Controlador (Servlet) recibe la petición GET/POST

3

Controlador invoca al Modelo (DAO) para procesar

4

Modelo accede a la BD y retorna datos

5

Controlador selecciona la Vista y envía datos

6

Vista (JSP) renderiza HTML y lo muestra al usuario

JavaServer Pages (JSP)

JSP es una tecnología que permite crear páginas web dinámicas usando Java. Con Jakarta EE 10, JSP 3.1 ofrece:

Expression Language (EL)

Sintaxis${producto.nombre}para acceder a datos sin scriptlets

JSTL (JSP Standard Tag Library)

Etiquetas como<c:forEach>,<c:if>para lógica sin Java

Directivas y Acciones

<%@ page %>,<%@ taglib %>para configuración

Objetos Implícitos

request,response,session,applicationdisponibles

Ejemplo: JSP con JSTL (Sin Scriptlets)

productos/lista.jsp

<%@ taglib prefix="c" uri="jakarta.tags.core" %>

    <c:forEach var="producto" items="${productos}">

            <td>$<c:out value="${producto.precio}" /></td>
        </tr>

</body>
</html>

JDBC - Java Database Connectivity

JDBC es la API estándar de Java para conectarse a bases de datos relacionales. En esta guía usamos PostgreSQL con JDBC 42.7.

Patrón DAO (Data Access Object)

El patrón DAO separa la lógica de acceso a datos del resto de la aplicación:

ProductoDAO.java — Insertar 
public class ProductoDAO {
    private static final String SQL_INSERT =         "INSERT INTO productos (codigo, nombre, precio, stock) VALUES (?, ?, ?, ?)";
    public boolean insertar(Producto producto) {
        // try-with-resources garantiza el cierre automático        try (Connection conn = ConexionDB.conectar();
        PreparedStatement ps = conn.prepareStatement(SQL_INSERT)) {
            ps.setString(1, producto.codigo());
            ps.setString(2, producto.nombre());
            ps.setBigDecimal(3, producto.precio());
            ps.setInt(4, producto.stock());
            int filasAfectadas = ps.executeUpdate();
            return filasAfectadas > 0;
        }
    catch (SQLException e) {
        Logger.getLogger(getClass().getName())                  .log(Level.SEVERE, "Error al insertar", e);
        return false;
    }
}
}

Records de JDK 21

Los records son una característica de JDK 14+ (estable desde JDK 16) que proporciona una sintaxis concisa para crear clases inmutables:

Clase Tradicional (50+ líneas) 

public final class Producto {
    private final int id;
    private final String codigo;
    private final String nombre;
    private final BigDecimal precio;
    // Constructor    public Producto(int id, String codigo,                     String nombre, BigDecimal precio) {
        this.id = id;
        this.codigo = codigo;
        this.nombre = nombre;
        this.precio = precio;
    }
// Getters    public int id() {
    return id;
}
publicString codigo() {
    return codigo;
}
publicString nombre() {
    return nombre;
}
publicBigDecimal precio() {
    return precio;
}
// equals(), hashCode(), toString()
// ... 30 líneas más
}

Record JDK 21 (1 línea) 

public record Producto(    int id,    String codigo,    String nombre,    BigDecimal precio) {
    // Constructor alternativo    public Producto(String codigo,                     String nombre,                     BigDecimal precio) {
        this(0, codigo, nombre, precio);
    }
}// Genera automáticamente:// - Constructor// - Getters (id(), codigo(), etc.)// - equals() y hashCode()// - toString()

Ventajas de los Records

Inmutabilidad:Los datos no pueden cambiar después de creados
Concisión:Menos código, menos errores
Seguridad:Thread-safe por diseño
Pattern Matching:Compatible con switch expressions
